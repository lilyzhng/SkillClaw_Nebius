Discover Skills → Skill Composition → Skill Creation

Robots can execute skills. They can't compose new ones.

Current systems chain existing skills by rules. SkillClaw trains robots to learn compositional grammar — generating new skills from primitives via RL.

01 — Core Thesis
RL environments don't just train policies — they can train skill composition itself
Traditional
  • Give the robot a task
  • RL trains a single policy
  • Robot learns to do this one thing
→
SkillClaw
  • Give the robot primitives + environment + reward
  • RL trains composition ability
  • Robot learns to create new skills from primitives

"The environment is a trigger, not a teacher" — Skill search is static (limited to existing skills). SkillClaw is generative (creates new ones).

Four Principles of Environment Design
Provide Building Blocks

Tidybot skills, SDK docs, tools — but don't tell it how to assemble

Tell It Right or Wrong

Sim verification + per-step criteria feedback

Don't Tell It How

Agent discovers decomposition → recomposition on its own

Don't Tell It Where

Agent uses bash grep/find/cat to discover context

03 — Training Plan
Skip SFT, Go Straight to GRPO

Key takeaway from OpenEnv hackathon: SFT degrades the base model. Instruct models already know tool calling format — GRPO directly teaches them to compose tools more efficiently.

APEX Sandbox + Tidybot Skill Catalog

Architecture reuses the OpenEnv hackathon's ApexEnvironment (bash sandbox + step/reset lifecycle), just swap the building blocks:

reset() → workspace with:
  skills/                  # Tidybot skill catalog
    ├─ tb-pick-up/scripts/main.py
    ├─ tb-center/scripts/main.py

  docs/                   # SDK guide, robot specs

step("cat specs.md")                 → task description
step("cat skills/tb-pick-up/...")    → discover primitive
step("cat > tb-clear-table/...")     → compose new skill
step("done")                        → reward
